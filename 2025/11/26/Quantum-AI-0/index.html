Ml-KEM算法安全性

关键点：

  • ML-KEM 的安全性直接归约到 MLWE 问题的困难性

  • MLWE 是 LWE 在模块结构上的推广

格问题

离散数学中的布尔格，和密码学中的格，不是一个格。

  • 在序理论中，一个“格”是指任意一个其中每两个元素都有唯一最小上界和唯一最大下界的偏序集。布尔格是这种“格”的一种特殊且重要的类型。
  • 在几何数论中，由于这种点阵结构也满足上述偏序集“格”的定义（如果我们用向量分量来定义偏序），所以也被称为“格”。

密码学中的“格”（Lattice）

这种“格”源于几何和线性代数。

  • 定义：一个格是由一组线性无关的向量（称为基向量）的所有整数系数线性组合构成的点的集合。

  • 公式化表达：给定一组基向量 (\mathbf{b}_1, \mathbf{b}_2, …, \mathbf{b}_n \in \mathbb{R}^m)，由它们生成的格是：
    [
    \mathcal{L} = { x_1\mathbf{b}_1 + x_2\mathbf{b}_2 + … + x_n\mathbf{b}_n \ | \ x_i \in \mathbb{Z} }
    ]

  • 直观理解：想象一个无限延伸的、规则排列的网格点阵（比如二维平面上的所有整数坐标点）。这些点阵就是一个格。基向量定义了网格的朝向和基本单元的形状。

  • 相关的“格问题”：在这些几何点阵上定义的困难计算问题，最著名的是：

    • 最短向量问题：在这个网格点阵中，找到一个离原点最近的非零点。
    • 最近向量问题：给定一个不在网格上的点，在网格上找到一个离它最近的点。
    • 学习伴随误差问题：可以看作是带噪声的最近向量问题。
特征 密码学中的“格” 布尔格
数学分支 几何数论、线性代数 序理论、抽象代数
本质 一个几何点集 一种代数结构
核心关系 向量之间的线性组合 元素之间的偏序关系
基本运算 向量加法、数乘 交、并、补
直观形象 无限延伸的网格点阵 (n) 维超立方体的哈斯图
相关的困难问题 最短向量问题、最近向量问题 无此类计算困难问题
主要应用 密码学（构建加密方案） 逻辑设计、集合论

“Module” 体现在：秘密和噪声是环上的向量，而非标量。它比 Ring-LWE 更灵活（可调整维数 (k, l)），比纯 LWE 更高效（利用环结构压缩数据）。

“Learning”体现在哪里？

这来自于机器学习的一个类比。想象一个线性回归模型：

  • 输入：样本 (A)
  • 权重：秘密 (s)
  • 带噪声的输出：(b = A s + e)

“学习”的任务是：给定许多 ((A, b)) 对，“学习”出隐藏的权重 (s)。
如果 (e=0)，这是简单的线性代数问题。但加上小的随机噪声 (e) 后，就变得计算困难。

在密码学中，“Learning” 就是指从这个带噪声的线性方程中恢复秘密的计算过程。

“Errors”体现在哪里？

“Errors”就是公式中的 (\mathbf{e})：
[
\mathbf{b} = A\mathbf{s} + \mathbf{e}
]

  • (\mathbf{e}) 是从一个窄分布（例如离散高斯分布，集中在 0 附近）中随机采样的。
  • 如果没有 (\mathbf{e})，那么 (\mathbf{b}) 就在由 (A) 生成的格中，我们可以直接用线性代数求出 (\mathbf{s})。
  • 但有了小的 (\mathbf{e})，(\mathbf{b}) 就只是接近格点，而不是正好在格上。这个小小的偏移使得问题变得非常困难。

错误的存在是安全性的核心。它将简单的线性问题变成了 conjectured 难解的抗量子计算问题。

ECDH 与 Ring-LWE 的对应关系

升级的核心是引入几何/代数结构：

特性 ECDH (椭圆曲线DH) Ring/Module-LWE
新基础结构 椭圆曲线群 (E(\mathbb{F}_p)) 多项式商环 (R_q = \mathbb{Z}_q[x]/(x^n+1))
核心思想 用曲线上的一个点来代替 (\mathbb{F}_p) 中的一个数 用环中的一个多项式来代替 (\mathbb{Z}_q^n) 中的一个向量
效率提升原理 一个椭圆曲线点 ((x, y)) 坐标在 (\mathbb{F}_p) 中，但整个点的标量乘法提供了比单个 (\mathbb{F}_p) 数模幂高得多的“安全密度”。 一个多项式有 (n) 个系数，但整个多项式的乘法提供了比 (n) 个独立 (\mathbb{Z}_q) 数更紧凑的运算结构。
安全密度 在 ~256 位的椭圆曲线上实现的安全强度，需要 ~3072 位的有限域 DH。 在 (n=256) 或 (512) 的 Ring-LWE 上实现的安全强度，需要 (n>1000) 的经典 LWE。
密钥大小 小 (一个曲线点，~32 字节) 小 (几个多项式，~1 KB)
带来的好处 密钥小、带宽低、计算快 密钥小、带宽低、计算快 (可利用 NTT)

这个共同的原理可以概括为：

从“非结构化”的抽象代数结构，迁移到“高度结构化”的抽象代数结构。

  • DH → ECDH：从有限域乘法群（结构相对简单）迁移到椭圆曲线加法群（具有丰富的几何和代数结构）。椭圆曲线群上的离散对数问题，被认为比在大小相近的有限域上困难得多。
  • LWE → Ring/Module-LWE：从随机整数向量的线性空间（非结构化）迁移到多项式环上的模（具有循环卷积等丰富的代数结构）。这种结构允许我们“打包”更多信息到一个运算中。
